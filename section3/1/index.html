if

If, czyli “jeżeli”. Podstawowa instrukcja warunkowa, której działanie najprościej porównać do działania trybów warunkowych np. w angielskim. Jeżeli zostanie spełniony warunek, coś się stanie, konkretnie:

int a = 3;

if (a < 5) {
a++;
}

// a = 4

Zmienna a na początku ma wartość 3. Instrukcja warunkowa mówi nam: “Jeżeli zmienna a jest mniejsza (nie równa) 5, dodam do niej 1”. W rezultacie po wykonaniu kodu zmienna a ma wartość 4. A co jeżeli warunek nie zostałby spełniony (np. zmienna a miałaby wartość 6)? Komputer pominąłby instrukcje zawarte wewnątrz instrukcji warunkowej (w naszym przypadku nie dodałby 1 do wartości a) i kontynuowałby dalej wykonywanie programu.

int b = 4;

if (b > 10) {
b = 5;
} else {
b = 15;
}

// b = 15

W tym wypadku nasz warunek nie został spełniony (4 nie jest większe od 10), więc komputer wykonał kod zawarty wewnątrz “else”. “Else” oznacza “w przeciwnym razie”, czyli instrukcje zawarte wewnątrz “else” będą wykonywane, kiedy warunek zawarty w “if” nie zostanie spełniony. Dlatego zmienna b po wykonaniu programu ma wartość 15.

int c = 7;

if (c == 8) {
c = 0;
} else if (c < 8) {
c = 10;
} else {
c = 1;
}

// c = 10

Tutaj widzimy bardziej rozbudowaną instrukcję warunkową w której mamy if, else if oraz else. Komputer najpierw sprawdzi czy zmienna c jest równa 8 (nie jest), po czym przejdzie do kolejnego testu - tym razem sprawdzi, czy c jest mniejsze od 8 (jest) i przypisze do zmiennej wartość 10, po czym zakończy wykonywanie instrukcji warunkowej. Nie przejdzie dalej, nawet gdybyśmy mieli jeszcze milion instrukcji “else if”. Komputer znalazł pasujący warunek, więc nie będzie szukał kolejnego.
